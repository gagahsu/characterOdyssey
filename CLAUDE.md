## Project Overview

**Character Odyssey** — A gamified multi-language phonetic learning app for children, covering Zhuyin (注音), English, Japanese (五十音), and Korean (韓文). The app supports multiple child profiles per account so siblings can each track independent progress.

## Tech Stack

| Layer | Technology |
|---|---|
| Frontend | Angular (SPA), CSS Grid, RxJS |
| Backend | Spring Boot (Java) |
| Database | PostgreSQL |
| Auth | Spring Security + JWT |
| Audio | howler.js + Audio Sprite |
| Stroke animation | Hanzi Writer (Chinese/Zhuyin), KanjiVG SVGs (Japanese), CSS `stroke-dasharray` (EN/KR) |
| Canvas tracing | HTML5 `<canvas>` overlaid on SVG |
| Audio generation | Python + `edge-tts` (offline batch script, run once in dev) |

## Architecture

### Frontend Module Boundaries

- **`CoreLearningModule`** — character tables, stroke animation, word-image cards
- **`GameModule`** — unlock map, quiz engine, timer logic (RxJS `interval`/`timer`)
- **`AuthModule`** — login, JWT storage, profile switcher
- **`SharedModule`** — audio service (howler.js wrapper), canvas drawing directive

Angular Router `CanActivate` guards enforce level locks: guards call the backend `/api/progress/{profileId}/unlocked` endpoint before activating a quiz route.

### State Management

A single `ProfileService` (or NgRx store slice) holds the **active child profile ID**. All components and HTTP calls read from it — never from local component state — so switching profiles mid-session stays consistent.

### Backend Data Model

```
User  1──* Profile  1──* LearningRecord
                              ├── char (字符)
                              ├── language (ZHUYIN|EN|JP|KR)
                              ├── correct (boolean)
                              └── timestamp
```

Spring Boot exposes REST endpoints under `/api/`; Spring Security validates JWT on every request and enforces that a user can only read/write records belonging to their own profiles.

### Audio Sprite Strategy

All phonemes for a given language are baked into a single `.mp3` sprite file (generated by the offline Python script). howler.js loads each sprite once on module init and plays the correct offset on click — eliminates per-click HTTP requests and the latency that causes when a child taps rapidly.

### Stroke Order Components

- **Zhuyin / Chinese**: use `HanziWriter` library directly; it ships stroke data and handles playback + trace detection.
- **Japanese**: load KanjiVG SVGs from `assets/svg/jp/`; animate `stroke-dashoffset` via JS.
- **English / Korean**: SVG paths from Noto Sans; animate with `stroke-dasharray` CSS + JS loop.

Canvas tracing overlay sits in a separate `<canvas>` element positioned `absolute` over the SVG, so SVG and drawing are independent layers.
